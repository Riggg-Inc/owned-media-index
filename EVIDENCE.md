# Evidence Rules

The Owned Media Index should be opinionated, but not hand-wavy.

Each entry should explain what kind of evidence supports it.

## Evidence Types

### Internal Production Data

Data from Riggg production workflows, customer programs, deliverable performance, distribution calendars, or operational outcomes.

Use when available, but remove private customer information unless permission exists.

### Teardown Observation

Analysis of public shows, episodes, posts, websites, YouTube channels, newsletters, or distribution patterns.

Use when a public example demonstrates a repeatable pattern.

### Platform Documentation

Official documentation from platforms such as YouTube, LinkedIn, podcast hosts, analytics providers, or search engines.

Use for technical requirements and platform-specific standards.

### Practitioner Observation

Experience-based judgment from operators, producers, strategists, editors, marketers, or customers.

Use honestly. Do not pretend observation is benchmark data.

### External Research

Credible reports, studies, benchmarks, or public datasets.

Use when claims depend on market behavior, audience behavior, or performance benchmarks.

## Evidence Labels

Use one of these labels:

- `hypothesis`
- `practitioner-observation`
- `internal-data`
- `external-research`
- `benchmark-backed`

## Public Safety

Do not publish:

- Confidential customer information.
- Private Airtable data.
- Private Slack messages.
- Unapproved customer performance metrics.
- Internal workflow secrets that should remain proprietary.

When using internal learning, generalize it into a public pattern.
