# Produce, Package, Publish, Prove, Preserve

The Owned Media Index uses a Riggg-owned open framework:

**Produce -> Package -> Publish -> Prove -> Preserve**

This is the operating model for turning expert conversations, webinars, livestreams, podcasts, and virtual event sessions into published owned media.

## Before the Framework: What You're Working With

Owned media splits into two things:

- **The asset** (what you made) — always owned. Video files, transcripts, clips, written posts. You created it; you own it regardless of where it gets published.
- **The distribution layer** (where you put it) — determines whether your *reach* is owned or rented.

The 5Ps describe what you do with both. See [`OWNED-MEDIA.md`](./OWNED-MEDIA.md) for the full definition, ownership tests, and asset inventory.

## 1. Produce

Create the source media.

Includes:

- Recording.
- Remote capture.
- Live production.
- Real-time production.
- OBS workflows.
- Audio and video standards.
- Hardware and room setup.

Quality question:

**Was the session captured in a way that creates clean, flexible source material?**

## 2. Package

Turn the source media into useful assets.

Includes:

- Long-form mastering.
- Titles.
- RSS descriptions.
- YouTube descriptions.
- Show notes.
- Chapters.
- Clips.
- Reels.
- Thumbnails.
- Quote graphics.
- Guest-share assets.

Quality question:

**Can someone understand, click, watch, listen, share, or reuse this asset without extra explanation?**

## 3. Publish

Get the assets into the right places.

Includes:

- Website-first canonical publishing.
- RSS distribution.
- YouTube publishing.
- LinkedIn publishing.
- Email.
- Social clips.
- Live premieres.
- Simulcast distribution.

Quality question:

**Did the media actually reach the channels and audiences it was created for?**

## 4. Prove

Measure whether the system worked.

Includes:

- Reach.
- Engagement.
- Watch/listen behavior.
- Conversion events.
- CTA performance.
- CRM attribution.
- Pipeline influence.
- Benchmarks.
- Scorecards.

Quality question:

**Can the team see what shipped, what worked, and what should change next?**

## 5. Preserve

Make the content compound.

Includes:

- Transcripts.
- Searchable archives.
- Vector memory.
- Content libraries.
- Theme extraction.
- Reuse patterns.
- AI and search visibility.

Quality question:

**Can the organization find and reuse what it already knows?**

## Why This Framework Exists

Most media workflows stop too early.

Some teams produce but do not package. Some package but do not publish. Some publish but do not prove. Almost everyone underinvests in preservation.

The Index exists to document the best patterns across the full chain.
