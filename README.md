# Owned Media Index

The **Owned Media Index** is an open, research-backed library of patterns, standards, tools, and benchmarks for turning expert conversations into published owned media.

Published by **Riggg Inc.**

## What Is Owned Media?

See [`OWNED-MEDIA.md`](./OWNED-MEDIA.md) for the core definition: the asset/distribution distinction, the ownership tests, a full asset inventory by 5P stage, and how Riggg relates to the PESO model.

> **tl;dr:** You own what you create. You do not own the pipe you send it through. The asset test and channel test determine what's owned and what's rented.

---

## The Framework

The Index is organized around the owned media operating model:

1. **Produce** - capture and create the source media.
2. **Package** - turn the session into usable assets.
3. **Publish** - distribute the assets across the right channels.
4. **Prove** - measure performance and business impact.
5. **Preserve** - store the content so it compounds over time.

This repository is not just a podcasting guide. It is a practical decision library for podcasts, webinars, livestreams, virtual events, and other episodic owned media programs.

## What Lives Here

The Index documents:

- Ranked title, description, hook, and metadata patterns.
- Distribution standards for website, RSS, YouTube, LinkedIn, email, and live premieres.
- Production standards for recording, live production, OBS workflows, and real-time production.
- Hardware and software options by use case.
- Benchmarks and scorecards for performance.
- Search, transcript, and vector-memory practices for long-term content reuse.

## Core Entry Type: Pattern

Most entries are patterns.

A pattern is a reusable answer to a production, packaging, publishing, measurement, or preservation problem.

Every pattern should answer:

1. **What is it?**
2. **Best for what?**
3. **Why does it work?**
4. **What does good look like?**
5. **When should you not use it?**
6. **What evidence supports it?**

## Repository Map

- `produce/` - recording, live production, real-time production, hardware.
- `package/` - titles, descriptions, show notes, clips, reels, thumbnails, quote graphics, chapters, guest assets.
- `publish/` - website, RSS, YouTube, LinkedIn, email, live premieres, simulcast.
- `prove/` - benchmarks, analytics, attribution, conversion events, scorecards.
- `preserve/` - transcripts, vector memory, content library, searchability, reuse patterns.
- `tools/` - software and tooling by production category.
- `schemas/` - structured schemas for patterns, tools, benchmarks, and distribution entries.

## Working Claim

Owned media should not die in a folder after the recording.

The best teams produce it intentionally, package it clearly, publish it consistently, prove what worked, and preserve the knowledge so every session compounds.
