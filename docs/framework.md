# Produce, Package, Publish, Prove, Preserve

The Owned Media Index is organized around a five-stage operating model.

## Produce

Create clean, flexible source media.

This includes recording, live production, real-time production, OBS workflows, audio/video quality, backup recording, and hardware setup.

**Quality question:** Was the session captured in a way that creates useful source material?

## Package

Turn the source media into usable assets.

This includes long-form mastering, titles, descriptions, show notes, chapters, clips, reels, thumbnails, quote graphics, and guest-share assets.

**Quality question:** Can someone understand, click, watch, listen, share, or reuse the asset without extra explanation?

## Publish

Get the assets into the right places.

This includes website-first publishing, RSS, YouTube, LinkedIn, email, social clips, live premieres, and simulcast distribution.

**Quality question:** Did the media actually reach the channels and audiences it was created for?

## Prove

Measure whether the system worked.

This includes reach, engagement, watch/listen behavior, conversion events, CTA performance, CRM attribution, pipeline influence, benchmarks, and scorecards.

**Quality question:** Can the team see what shipped, what worked, and what should change next?

## Preserve

Make the content compound.

This includes transcripts, searchable archives, vector memory, content libraries, theme extraction, reuse patterns, and AI/search visibility.

**Quality question:** Can the organization find and reuse what it already knows?
