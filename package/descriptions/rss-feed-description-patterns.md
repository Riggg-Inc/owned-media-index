# RSS Feed Description Patterns

## What It Is

Reusable patterns for podcast RSS episode descriptions.

## Best For

- Apple Podcasts.
- Spotify.
- Amazon Music.
- Pocket Casts.
- Podcast apps that display RSS metadata.

## Why It Works

RSS descriptions need to help listeners decide whether an episode is worth their time. They also need enough context to support search, accessibility, and downstream repurposing.

## Example Patterns

1. **Problem -> Guest Credibility -> What You Will Learn -> CTA**
2. **Big Question -> Conversation Summary -> Key Takeaways -> Links**
3. **Audience Pain -> Practical Breakdown -> Resources**
4. **Guest Story -> Lessons -> Application**
5. **Topic Thesis -> Three Takeaways -> Subscribe/CTA**
6. **Timely Context -> Expert Perspective -> Why It Matters**
7. **Case Setup -> What Happened -> What Listeners Can Use**
8. **Contrarian Hook -> Nuanced Explanation -> Next Step**
9. **Operator Lesson -> Mistakes Avoided -> Playbook**
10. **Short Summary -> Bulleted Takeaways -> Guest Links**

## Quality Bar

A strong RSS description should include:

- A clear first sentence.
- Guest name and relevance where applicable.
- What the listener will learn.
- 3 to 5 key takeaways when appropriate.
- Links or resources when available.
- Clear CTA.

## When Not To Use

Avoid long descriptions when the podcast app truncates heavily and the first sentence does not carry the value.
