# Simulcast Standard

## What It Is

A standard for broadcasting a Program or Session live across multiple platforms.

## Best For

- Live shows.
- Webinars.
- Launch events.
- Virtual event sessions.
- High-value thought leadership conversations.

## Why It Works

Live streaming turns distribution into an event. Simulcasting lets a Program meet the audience where they already spend attention.

## Required Decisions

- Primary live destination.
- Secondary destinations.
- Horizontal and/or vertical output.
- Stream title and description per platform.
- Thumbnail or holding graphic.
- Guest/host promotion plan.
- Recording backup.
- Post-live replay plan.

## Quality Bar

The stream should feel intentional on every destination, not like a single broadcast blindly sprayed everywhere.

## When Not To Use

Avoid simulcast when:

- The session is private.
- The content requires heavy editing before release.
- The guest is not prepared for live participation.
- Platform-specific audience context would make one generic stream confusing.
